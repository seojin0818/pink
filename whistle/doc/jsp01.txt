# jsp01

JSP란?
==> 
	Web 서버가 클라이언트에게 응답하기 위해 필요한 문서
	(주로 HTML 문서)를 만들어주는 언어의 일종
	
	Web Server가 가진 엔진이 실행할 수 있는 언어는 따로 있음
	이것을 우리는 Server Side Page(SSP)라고 말함
	
	종류 ]
	
		대표적으로
		
			ASP(Active Server Page)
				==> Visual Basic
			ASP.NET
				==> C#
			PHP
				==> CGI
			JSP
				==> Java
		
		결론적으로
		우리가 배우는 앞으로의 내용은
		서버가 실행해서 클라이언트에게 응답하는
		응답 문서를 만들기 위한 언어를 배우는 것
		
		즉, 서버가 실행할 수 있는 프로그램을 만드는 것이고
		그 결과는 문서(파일로 탄생하게 되며)
		그 파일의 결과는 반드시 클라이언트의 웹 브라우저를 이용해서 테스트 해야만 함
		
	Web Server
	==> Http 프로토콜에 의해서 클라이언트의 요청을 받아서 해결하는 서버를 지칭하는 단어
	
		예 ]
		
			아파치, 톰캣, 레진, IIS, 엔진X, .....
			
		이 중 JSP처럼 클라이언트의 요청에 따라서 동적으로 응답 문서를 만들어주는 
		어플리케이션을 실행할 수 있는 서버를 Web Application Server라고 부름 ==> 줄여서 WAS
		
		참고 ]
		
			프로젝트를 만들고
			서버를 만든 후
			서버에 프로젝트를 추가한 후 확인해야 할 내용
				server.xml
				web.xml
			을 필요한 내용으로 되어있는지 확인해줘야 함
			
		참고 ]
		
			프로젝트에서 사용할 파일들은 인코딩을 UTF-8로 설정해두는 것이 좋음

프로젝트의 폴더 구조

	src					: 프로젝트의 기본 폴더. 프로젝트에 관련된 파일들은 이 폴더 하위에 들어가야 함
	src/main			: 실제 프로젝트 소스 코드들이 작성되는 곳
	src/main/java		: 프로젝트의 자바 소스코드가 작성되는 곳
	src/main/webapp		: 프로젝트의 뷰에 관련된 문서들이 저장되는 곳
							==> 실제 JSP, HTML, js, css 등 응답에 필요한 문서가 작성되는 위치
							
								META-INF : 우리는 이 폴더를 데이터베이스 설정에 관련된 문서가 위치하는 폴더로 사용할 것
								WEB-INF : 서버에 관련된 설정들 또는 자바클래스가 뷰를 저장했다가 부르는 폴더
								==> 이 두 폴더는 클라이언트가 접근할 수 없는 폴더
									서버만 접근할 수 있는 폴더
									
	참고 ]
	
		XML 파일
			Extensible Markup Language의 약자
			의미는 확장 가능한 마크업 언어 (마크업 언어 : 태그로 이루어지는 문서들)
			
			태그를 이용해서 파일의 내용을 구분하도록 만들어지는 문서를 이야기하고
			그 중 하나가 HTML 문서
			
			이 때 HTML의 경우는 사용하는 태그가 이미 결정되어 있지만
			XML의 경우는 사용자가 태그의 종류를 정한 후
			그 태그를 이용해서 문서를 만드는 것을 말함
			
			이 문서는 주로 프로그램의 환경 설정을 하기 위한 목적으로 많이 사용됨
			
			프로그램은 프로그램을 실행하기 전
			먼저 XML 파일을 읽은 후
			그 파일의 태그를 분석함 ==> 이 작업을 우리는 "파싱" 시킨다고 표현
			그리고 그 파싱된 결과를 프로그램에서 이용하는 것이 됨
			
			***
			결론적으로 어떤 프로그램의 실행 환경(시작 환경)을 바꾸고 싶으면
			우리는 XML 파일에 그 환경을 설정하는 부분을 찾아서 수정하면 됨
			
			참고 ]
			
				파싱을 하기 위해서
				그 XML 문서에 있는 태그의 의미를 알아야 함
				XML 문서를 만드는 사람이 태그의 순서, 구조, 의미를 지정하는 문서가 따로 존재함
				이 문서를 "DTD" 또는 "스키마 문서"라고 함
